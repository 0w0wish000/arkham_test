# 11 · 完整引擎開發清單(Beyond-Lite / 多劇本)

> 目前是 **lite 引擎**:單一硬編場景(Spreading Flames lite)、只做移動/調查/戰鬥/閃避/交戰/技能檢定/回合、`PLAY_CARD`/`ACTIVATE` 未實作。
> 本清單是把它升級成「**完整規則引擎 + 全內容 + 戰役系統**」的工作盤點,**依功能(epic)分組**。設計依據:[02 系統設計](02-design-spec.md)、[05 規則引擎規格](05-rules-engine-spec.md)、[04 路線圖/風險](04-roadmap-risks.md)。
>
> 規模:**S**(數天)/ **M**(1–2 週)/ **L**(數週)/ **XL**(數月～持續)。狀態:☐ 未開始 · ◐ 部分 · ☑ 完成。
> **相依鏈:B(能力引擎)是樞紐** —— 它擋著 C(全卡)與大半 fidelity;A(場景資料化)擋著 D(戰役)。

## 🧭 建議建置順序(相依驅動)

```
1. B 能力引擎原型(B1→B2→B3→B4)      ← 最高風險,先做原型驗證可行性
2. A 場景資料化(A1→A2)               ← 與 B 平行可動
3. C1–C2 卡牌 DSL + play/activate     ← B 之上,先跑通幾張代表卡
4. E1–E2 回合/階段 fidelity           ← 接上 B3 的能力視窗
5. D 戰役系統(D1→D2→D3 修 maxXp→D4/D5)← A + 部分 B 之後
6. C6 逐盒實作卡池 + G1/G2 資料/翻譯   ← 持續擴張(核心→擴充)
7. F 牌組驗證;F4/G4 待帳號/商業層
```

**里程碑(對齊 docs/04)**
- **M-α 垂直切片**:B 原型 + A 資料化 + 3–5 張代表卡 + 一個資料化劇本可玩到底。
  - 🧪 **已有第一步:測試沙盒**(`campaignKey=sandbox`)—— 引擎已實作 `PLAY_CARD` + 一組手寫效果的特殊卡(緊急補給 +3 資源、放大鏡 +1 智力/進檯面、大砍刀 +1 戰傷、靈光一閃免檢定取線索、急救治療),線索充足 + 弱敵(訓練假人)+ 高資源,專供**試牌組 / 試特殊卡**。前端手牌可點擊打出、顯示費用與檯面。驗證見 `e2e/sandbox-e2e.mjs`(14 斷言)。這是 §B/§C 的迷你切片;正式版改走能力引擎 + 效果 DSL。
  - 🎮 **玩法批次(已完成)**:①`ChaosBag.forDifficulty`(EASY 15 / STANDARD 16 / HARD 16 / EXPERT 17 顆,§A5 lite 版)②**C-lite 牌組管線**:`CardCatalog`(選卡器全卡池的卡名→型別/費用/圖示)+ SET_DECK 卡名 → 洗牌 → 開局抽 5 → 整備抽 1(空堆洗回 +1 恐懼)—— 建牌真的進遊戲 ③**END_TURN 屏障化**(§E1 lite):「我打完了」全員完成才結算,`force` 可強制;防一人手滑蒸發全桌行動。驗證見 `e2e/gameplay-e2e.mjs`(14 斷言)。
  - 📥 **規則符合度移植批次(已完成)**:自 `fix/rules-flow-conformance` 分支挑相容內容移植(該分支的逐人回合制與「隊友投入不限 1 張」**不採**—— 前者違反自由順序設計、後者違反官方 p15):①**官方基本行動補齊** —— 抽牌 / 取資源 / 撤退(p14,豁免趁隙、不減人數縮放);交涉=卡牌能力(待 C1)②**個別淘汰制**(p41)—— 被擊敗/撤退只退場(線索掉落地、威脅區敵人解交戰留場),全員退場才「未達成結局」;視圖加 `elimination` 欄位,前端顯示觀戰狀態 ③**人數縮放鎖定** `GameState.playerCount`(開局鎖定,撤退不減)④**場上毀滅計數**(p18)—— 密謀門檻比對 agenda+敵人+地點卡上的毀滅 ⑤**mulligan**(p20)⑥**趁隙攻擊補全** —— 打牌 / 交戰 / 抽牌 / 取資源皆引發。驗證見 `PortedConformanceTest`(6 測試)+ 全套 e2e 迴歸。
