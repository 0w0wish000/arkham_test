# 07 · 區域網路連線(LAN)多人

**任何一位玩家都能當「主機」跑伺服器 —— 誰開都行**,和遊戲內的「首席調查員(lead)」完全無關。

## 架構(一句話)
一台跑**伺服器**(Java,port 8080);每位玩家用**瀏覽器**玩(前端 port 5173,把 WebSocket 轉到伺服器)。**同一房間(room)= 同一場**;每人用不同 `?inv=` 挑調查員。

```
   玩家A 瀏覽器 ┐
   玩家B 瀏覽器 ┼─(區網)─▶  主機那台:前端 5173  ──▶  伺服器 8080(規則引擎)
   玩家C 瀏覽器 ┘
```

## 前置(每台電腦裝一次)
- **JDK**:理想 21;較舊版也行(Gradle 會自動下載 21 toolchain)。只有跑**伺服器**的那台一定要。
- **Node.js 18+**(跑**前端**的那台要;**要跑 e2e 測試則需 Node 22+**,因為用到內建 WebSocket)。
- 首次啟動會下載 Gradle / JDK / npm 相依,需要**網路**;之後可離線區網玩。

## 🪟 Windows(主要環境)

### 方法一(最簡單):主機一台跑兩個,大家用瀏覽器
**主機**(誰都行)——直接**雙擊**這兩個檔(或在終端機執行):
```bat
start-server.bat      :: 視窗 A:會印出你的區網 IP,例如 192.168.1.50
start-client.bat      :: 視窗 B
```
> 也可以只雙擊 **`run-dev.bat`** 一鍵開這兩個視窗。

**所有人(含主機自己)** 用瀏覽器開,各挑不同調查員:
```
http://192.168.1.50:5173/?inv=joe_diamond     # 你
http://192.168.1.50:5173/?inv=daniela          # 隊友
```

### 方法二:每人各自跑 client
**主機**:雙擊 `start-server.bat`(記下印出的 IP)。
**每位玩家**在自己電腦開終端機:
```bat
start-client.bat 192.168.1.50      :: ← 主機的 IP(不是 localhost)
:: 然後開 http://localhost:5173/?inv=daniela
```

---

## 🍎 macOS / Linux
同樣兩種方法,換成 `.sh`:
```bash
./start-server.sh                 # 主機:印出區網 IP
./start-client.sh                 # 同機:連本機伺服器
./start-client.sh 192.168.1.50    # 別機:連主機 IP
# 或一鍵:./run-dev.sh
```

## 網址參數
| 參數 | 用途 |
|---|---|
| `?inv=joe_diamond` / `?inv=daniela` | 挑調查員(目前情境有這兩位) |
| `?room=xxx` | 開不同房間;**同房才在同一場遊戲**(預設 `demo-session`) |

## 存檔 / 續玩(重開載入)
- 遊戲中任一玩家按「💾 保存並離開」→ 全員跳出「是否存檔?」→ 只要有人同意即存檔;伺服器把**狀態文本 + 出牌紀錄**複製到**每位玩家本機**(瀏覽器 localStorage,每房一份)。
- 每回合結算時伺服器也會**自動存檔**一次(落地在主機的 `saves/<房號>.json`)。
- **重開續玩**:大家重新連進**同一房間**後,本機若偵測到該房存檔會問「由你載入續玩?」——**只需一人**點是,伺服器就用那份存檔重建對局、廣播給全員。詳見 [docs/08 §6.5](08-save-and-provenance.md)。

## ✅ 端到端測試(e2e,邊改邊修用)
一鍵驗證「伺服器 + 協定 + 引擎 + 存檔/續玩 + 前端建置」整條是否還健康:

- **Windows**:雙擊 `e2e.bat`(或終端機 `node e2e\run.mjs`)
- **macOS / Linux**:`./e2e.sh`(或 `node e2e/run.mjs`)

它會:①建置並啟動伺服器(以**真實 WebSocket 握手**判定就緒)→ ②用**兩個客戶端**跑完整流程(join → 調查〔多人投入屏障〕→ 移動〔揭示+生怪〕→ 回合推進 → 全員投票存檔 → 續玩還原)→ ③前端 `tsc` 型別檢查 + 打包 → ④**可靠收掉伺服器**(等埠釋放,必要時硬殺)。全過離開碼 0。

常用開關(環境變數):
| 變數 | 作用 |
|---|---|
| `E2E_SKIP_CLIENT=1` | 只跑協定 e2e,不建前端(迭代快) |
| `E2E_NO_SERVER=1` | 沿用已在跑的伺服器,不自己起 |
| `E2E_SKIP_PROTOCOL=1` | 只建前端,不跑協定 e2e |

> 只跑協定測試、想指定伺服器:`node e2e/protocol-e2e.mjs ws://192.168.1.50:8080`

## 疑難排解
- **連不上 / 隊友連不進來**:主機防火牆要放行 **8080 / 5173** 的區網連入。
  - Windows:第一次啟動時若跳出「Windows Defender 防火牆」提示,勾**私人網路**並允許。手動:控制台 → Windows Defender 防火牆 → 進階設定 → 輸入規則 → 新增 8080 / 5173(TCP)。
  - macOS:系統設定 → 網路 → 防火牆(測試可暫時關閉)。
- 所有人要在**同一區網**(同一台 Wi-Fi / 交換器)。
- **主機 IP**:腳本會自動印;手動查 → Windows `ipconfig`(找「IPv4 位址」)、macOS `ipconfig getifaddr en0`。
- **`start-*.bat` 秀亂碼**:那是中文說明字;不影響執行。視窗已自動 `chcp 65001` 切 UTF-8,若仍亂碼可改用 Windows Terminal。
- **8080 被占用 / 伺服器啟動失敗**:多半是上一個伺服器沒關乾淨。關掉舊視窗即可;e2e 協調器會自動清掉殘留的半死伺服器。

## 正式版會更簡單
把前端 `npm run build` 後,交給 Java 伺服器用**同一個 port(8080)** 一起服務靜態檔 → 大家只要開 `http://<主機IP>:8080`,連 5173 都不用。屆時「主機跑一支程式、其他人開一個網址」即可。
